
Lets Suppose, everything is handled or executed on a single system
Writing the program, 
testing the program,
Deployment of the application,
and all the client is executing the already developed application on the SAme Single System.

So, The performance of the System will not be good. 

SAP divided their System into 3 Different Parts for different Purpose and all these Systems are connected together and mainly handled By the BASIS.

- **Development SYSTEM** (800)
	- Developer has the Access of this System. On this System, We develop Our Product.

- **Quality SYSTEM**
	- This system is dedicated for the Testing purpose where your Testing team will test the application developed by the ABAP Team. 

- **Production SYSTEM**
	- This System is where your  End Users will finally execute the Application and they will do the final Testing.
	- Also your Client Will run the developed App on this SYStem.

**Transport request**

the transport Request Is a mechanism which is used to transport objects from One system to another System.
>> i.e. From the development System -> Q -> P
>> When we develop our Application, we will Request the BASIS team to import the applications, TR requests from Development system to Quality System for testing purposes. 
> It is very very Crucial to store all our developed application {i.e. programs, databases, objects, etc } on the transport Requests.
> If Your Developed Application will be aligned with the TR, then you can transfer it from one system to another System.
